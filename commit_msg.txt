fix(android): render GIF as window background to eliminate shake

Moves giphy.gif out of the MAUI view hierarchy entirely and renders it
as the Android Activity window background using AnimatedImageDrawable
(API 28+). Animation frames now call invalidate() only on the Window
decor-view background drawable — completely below MAUI's compositing
layer — so MAUI layout is never touched during animation.

Changes:
- Resources/Images/giphy.gif → Resources/Raw/giphy.gif (MauiAsset)
  so it is accessible via Android Assets.Open() at runtime
- Platforms/Android/MainActivity.cs: override OnCreate, read GIF bytes,
  wrap in ByteBuffer, decode via ImageDecoder (API 28+ guard + try/catch
  fallback), set Alpha=127 (50% opacity), call Start(), set as
  Window background drawable
- Views/MainPage.xaml: remove AbsoluteLayout + Image GIF block;
  set ContentPage and root Grid BackgroundColor=Transparent so the
  native window background shows through; BoxView gradient overlay kept

Pre-API-28 devices fall back gracefully (dark background only).
